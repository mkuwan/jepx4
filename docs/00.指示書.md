### 1.1 背景・目的

**背景**

*   現行運用は **Web 画面操作＋RPA** で入札/取消/照会/約定参照を実施。
*   JEPXがWeb画面を廃止し、API連携のみとなるため、これまでのRPA処理は使用できなくなる予定。
*   JEPX 公開 API 仕様（翌日＝**DAH**、時間前＝**ITD/ITN**）の利用を前提に、**RPA/手作業を廃止**し API 連携へ移行。
*   RPAで行っていた、計画値ファイルの読み込み、入札、入札照会、入札削除、約定結果比較などの処理をAPIで行う
*   バッチ処理で自動処理を行って読み込みから入札、結果取得などを行えるようにする
*   状況(ステータス)、入札削除、照会、比較、レポートなどは画面から操作、確認できる必要がある

**目的**

*   CSV/Excel 取込 → 検証 → API 実行 → 結果保存/監査 → レポートまでを **自動化**。
*   **冪等性・再送・証跡・BCP** を確保し、実運用に耐える仕組みを整備。
*   照会、削除、レポート、再入札などは画面から行う必要がある。

### 1.2 スコープ（業務範囲・システム化範囲）

*   **対象市場**
    *   翌日市場（DAH）：1 日 48 区分の電力量を前日入札。
    *   時間前市場（ITD）：当日内の分単位調整／情報通知（ITN）。
*   **対象業務**
    *   翌日市場（DAH）
    入札、入札削除要求、入札照会、約定参照、清算照会。
    *   時間前市場（ITD）
    入札、入札削除要求、入札照会、約定参照、清算照会、市場情報通知。

*   **対象外**
    *   会員サイトの手動操作（GUI 操作）、帳票レイアウト詳細（CSV/Excel/PDF 出力は対象）、公開仕様外 API。

### 1.3 現状（As-Is）

*   RPA依存


### 1.4 To-Be（あるべき業務像）

*   雛形（CSV/Excel）取込 → **API 実行→照会→出力** の定常フロー。
*   エラー時の再送・差替え・ロールバック手順（運用ガイド）を標準化。
*   時間前市場はすべてExcelからrequestされ、ウェブサーバーのアプリのAPIで受信、それをJEPXのAPIに送信する
*   翌日市場は基本的にバッチ処理で自動実行される
*   バッチ、ウェブサーバー(API)はpython(Django)で作成する
*   インフラ構成はバッチサーバーの前にFW、中継サーバーを経由してAWS Direct ConnectでAPIと連携する
*   インフラはクライアントで用意する
*   AWS環境を利用し、サーバーはVPCで構成する
*   ExcelファイルはVBAで取引情報を送る。その際に認証は不要でhttp requestのみ
*   Excelファイルで時間前市場の照会、市場情報取得もできる
*   翌日市場ではバッチで稼働する処理ではGraph APIを使用してSharePoint上のファイルを取得し、そのファイル(csv/excel)の内容に従って入札を行う

### 1.5 業務要件（Business Requirements）

*   **BR-01 目的/KPI**：定常日 入札失敗 0（9時までに完了）。
*   **BR-02 範囲**：翌日・時間前の入札/取消/照会/約定/集計。
*   **BR-03 RACI**：入力（運用）／実行・証跡保管（システム）。
*   **BR-04 フロー**：定常・例外の 2 系列。
*   **BR-05 データ**：CSV/Excel・PDF（Base64）ログ保存。
*   **BR-06 司令塔画面**：進捗・アラート・再送・監査ログ参照。
*   **BR-07 監査/BCP**：長期保管、DR 演習、手順整備。

### 1.6 機能要件（Functional Requirements）

*   **FR-01** 入札データ取込（CSV/Excel）SharePoint連携
*   **FR-02** バリデーション（required/format/range/unit/code/schedule/inconsistency）
*   **FR-03** 送信電文生成（JSON/gzip、ヘッダ SIZE 厳密）
*   **FR-04** API 実行（DAH/ITD/ITN）
*   **FR-05** 入札取消
*   **FR-06** 照会（入札・結果・市場情報）
*   **FR-07** 約定参照・集計（PDF Base64 対応）
*   **FR-08** 実行制御（二重送信防止）
*   **FR-09** 監視・通知（閾値・アラート）
*   **FR-10** 出力（CSV/Excel/PDF）
*   **FR-11** ログ出力（API I/O・操作ログ）
*   **FR-12** 運用ツール（再実行・テスト送信）
*   **FR-13** ウェブサーバー機能(ExcelとのAPI通信)

### 1.7 非機能要件（NFR）

*   処理性能（9時までに完了）、冪等・再送、照会、ログ、セキュリティ準拠。
