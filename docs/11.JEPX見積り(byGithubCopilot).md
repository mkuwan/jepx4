# JEPX API連携システム 開発見積書

**作成日**: 2026年3月9日  
**見積もり方式**: ファンクションポイント（FP）法  
**単位**: 人日

## 1. エグゼクティブサマリー

### 1.1 見積もり概要

本見積書は、JEPX（日本卸電力取引所）API連携システムの開発に必要な工数をファンクションポイント（FP）法により算出したものです。

| 項目 | 合計 |
|------|------|
| **総ファンクションポイント（UFP）** | 1,285 FP |
| **調整済ファンクションポイント（AFP）** | 1,413 FP |
| **総開発工数** | **236.7 人日** |

### 1.2 プロジェクト期間
- 基本設計～詳細設計: 40 人日
- 実装: 135 人日
- テスト: 51 人日
- リリース準備・移行: 10.7 人日

### 1.3 主要機能
1. **翌日市場（DAH）バッチ処理システム**
   - SharePoint連携による入出力
   - JEPX API通信（入札、照会、削除、約定、清算）
   - 自動実行・監視・リカバリ
   
2. **時間前市場（ITD）Web APIシステム**
   - Django RESTful API
   - Excel VBA連携
   - リアルタイム配信中継（ITN）
   
3. **共通基盤**
   - JEPX Socket通信プロトコル実装
   - TLS1.3暗号化通信
   - 監査ログ・エラーハンドリング

## 2. 前提条件と制約事項

### 2.1 開発方針
- **ステートレス設計**: データベース（RDB/NoSQL）は使用しない
- **UIはExcelのみ**: Webブラウザ向けGUI画面は作成しない
- **証跡はログファイル**: すべての操作履歴はログに記録

### 2.2 対象範囲
- ✅ 翌日市場（DAH）の通常入札のみ（**ブロック入札は対象外**）
- ✅ 時間前市場（ITD）の通常入札のみ（**ブロック入札は対象外**）
- ✅ 市場情報通知（ITN）の受信と配信
- ❌ ブロック入札機能は実装しない
- ❌ JEPX会員サイトGUI操作の代替は実装しない

### 2.3 技術スタック
- プログラミング言語: Python 3.11+
- Webフレームワーク: Django (ASGI/Uvicorn)
- 連携: Microsoft Graph API (SharePoint)
- インフラ: AWS EC2, Direct Connect経由JEPX接続
- スケジューラ: JP1 (バッチ処理)

### 2.4 成果物
- ソースコード（Python）
- 設定ファイル（YAML/JSON）
- 基本設計書
- 詳細設計書
- テスト仕様書・結果報告書
- 運用手順書

## 4. ファンクションポイント（FP）分析

### 4.1 FP分析の方針

#### 4.1.1 トランザクション機能（TF）
- **外部入力（EI）**: ユーザーからのデータ入力、ファイル取込
- **外部出力（EO）**: 複雑な計算・加工を伴う出力
- **外部照会（EQ）**: 単純なデータ取得・表示

#### 4.1.2 データ機能（DF）
- **内部論理ファイル（ILF）**: アプリケーション内で管理するデータ（設定ファイル等）
- **外部インターフェースファイル（EIF）**: 外部システムのデータ参照

#### 4.1.3 複雑度の定義
- **低**: 3-4 FP（単純処理、項目数少、単一パターン）
- **中**: 5-7 FP（標準処理、項目数中程度、複数パターン）
- **高**: 8-10 FP（複雑処理、項目数多、多様なパターン、エラー制御複雑）
