## 1. テスト実行環境

### 1.1 前提条件

| 項目 | 要件 |
|------|------|
| Python | 3.12 以上 |
| 仮想環境 | `jepx_project/.venv` を使用すること |
| 依存ライブラリ | `requirements.txt` に記載のパッケージがインストール済みであること |

### 1.2 環境構築手順

```powershell
# 1. プロジェクトディレクトリに移動
cd N:\antigravity\jepx4\jepx_project

# 2. 仮想環境の作成（未作成の場合）
python -m venv .venv

# 3. 依存ライブラリのインストール
.venv\Scripts\python.exe -m pip install -r requirements.txt
```

### 1.3 テスト実行方法

> [!CAUTION]
> **必ず仮想環境の Python を使用してください。**
> システムの Python (`C:\Users\...\Python312\python.exe`) を使用すると
> `ModuleNotFoundError: No module named 'django'` が発生します。

#### 全テスト一括実行（推奨）

```powershell
cd N:\antigravity\jepx4\jepx_project
.venv\Scripts\python.exe -m unittest discover -s tests -v
```

#### 個別テストファイルの実行

```powershell
cd N:\antigravity\jepx4\jepx_project
.venv\Scripts\python.exe -m unittest tests.test_protocol -v
.venv\Scripts\python.exe -m unittest tests.test_exceptions -v
.venv\Scripts\python.exe -m unittest tests.test_validators -v
.venv\Scripts\python.exe -m unittest tests.test_modules -v
```

#### 特定テストクラス/メソッドの実行

```powershell
# 特定クラスのみ
.venv\Scripts\python.exe -m unittest tests.test_validators.TestVolumeAutoCorrection -v

# 特定メソッドのみ
.venv\Scripts\python.exe -m unittest tests.test_protocol.TestBuildPacket.test_packet_structure -v
```

### 1.4 テスト結果の見方

```
test_packet_structure (test_protocol.TestBuildPacket.test_packet_structure)
SOH + Header + STX + gzip(JSON) + ETX の構造を検証 ... ok      ← 成功
```

- `ok` : テスト成功
- `FAIL` : アサーション失敗（期待値と実値の不一致）
- `ERROR` : テスト実行中に予期しない例外が発生

最終行に `Ran XX tests in X.XXXs` と `OK` が表示されれば全テスト合格。
