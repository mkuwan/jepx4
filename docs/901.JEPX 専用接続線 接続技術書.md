本書は、一般社団法人日本卸電力取引所通信線利用規約に則り、JEPX 専用接続線の利用について定める。  
本書で通信線利用規約第7条第3項の指定する通信方法を定義する。

# 1. 通信方法

JEPX システムとの通信は、Socket 通信と呼ばれる TCP/IP 通信を用います。通信は TLS1.3 で暗号化します。  
通信先の IP アドレス/ポート番号は、本取引所から通知したものを利用する。  
各サーバ（非化石、電力）毎に 1 つの IP アドレスのみアクセスを許可（IP アドレス制限）。

- 1 通信線 1 サーバあたりの動作 Socket 上限：**5 + 1（配信用は 1）**

# 2. 接続手法

## 2.1 事前用意
- 本取引所が配布するルート証明書を格納する  
  （サーバ証明書の検証を行わない場合は不要）

## 2.2 接続
- 通知された IP アドレス/ポート番号を指定して Socket を生成  
- 生成時に TLS 暗号化ハンドシェイクが行われる

## 2.3 通信

### 一般通信
- API 仕様書に従い Socket 書き込み／読み込みを行う
- **3 分間通信が無い場合、サーバ側 Socket が切断される**

### 配信通信
- API 仕様書に従い Socket 書き込み後、配信が開始される
- 通信が無くてもサーバ側からは切断されない

## 2.4 切断
- Socket 切断命令（close）で通信終了

# 3. 通信フォーマット

通信フォーマット：

```
SOH
ヘッダ部
STX
ボディ部
ETX
```

## 3.1 ヘッダ部
- 文字コード：ASCII
- 「項目名=値」形式

### 3.1.1 要求時ヘッダ情報

| 項目 | 説明 | 形式 |
|------|------|------|
| MEMBER | 会員 ID（試験時は 9999） | 英数字 4 桁 |
| API | 機能番号（API 仕様書参照） | 英数字 7 桁 |
| SIZE | ボディ部サイズ（Byte 数） | 数字 |

例：  
`MEMBER=0841,API=ABC1234,SIZE=421`

### 3.1.2 受信時ヘッダ情報

| 項目 | 説明 | 形式 |
|------|------|------|
| STATUS | 電文ステータス | 数字 2 桁 |
| SIZE | ボディ部サイズ（Byte 数） | 数字 |

例：  
`STATUS=00,SIZE=254`

#### 電文ステータス一覧

| STATUS | 内容 |
|--------|------|
| 00 | 正常処理（機能エラーでも電文は 00） |
| 10 | 電文フォーマット異常 |
| 11 | 会員 ID に利用権限なし |
| 19 | サーバシステム異常 |

## 3.2 ボディ部
- gzip で圧縮

# 4. システム（共通）API

## 4.1 ソケット接続時間延長（SYS1001）

| 項目 | 内容 |
|------|------|
| API 名 | SYS1001 |
| 概要 | 3 分間通信が無いと切断されるため、その切断を回避するリクエスト |

## 4.2 リクエストボディ部
- JSON 形式  
- 項目なし

## 4.3 レスポンスボディ部
- JSON 形式

| 項目名 | キー | 型 | 説明 |
|--------|------|----|------|
| 結果 | status | 文字列 | "200"（成功） |
| 結果情報 | statusInfo | 文字列 | "Socket Expiration Time Extension" |

例：

```json
{
  "status": "200",
  "statusInfo": "Socket Expiration Time Extension"
}
```
